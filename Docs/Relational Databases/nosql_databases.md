# What Is a NoSQL Database? (ฐานข้อมูล NoSQL คืออะไร?)

> 🗄️ **แหล่งที่มา:** [IBM — What Is a NoSQL Database?](https://www.ibm.com/think/topics/nosql-databases)

## 1. NoSQL คืออะไร?
**NoSQL (Not Only SQL หรือ Non-SQL)** คือแนวทางการออกแบบฐานข้อมูลที่ใช้จัดเก็บและดึงข้อมูลโดย **ไม่ใช้โครงสร้างแบบตาราง (Tabular)** เหมือนในฐานข้อมูลเชิงสัมพันธ์ (Relational Database) แบบดั้งเดิม
*   **Schema-less:** ไม่ต้องกำหนดโครงสร้างข้อมูลล่วงหน้าให้เป๊ะ ทำให้มีความยืดหยุ่นสูงมาก
*   **Distributed Database:** ข้อมูลถูกเก็บกระจายไว้ในหลายๆ เซิร์ฟเวอร์ (Distributed) ทำให้ทนทานต่อความเสียหาย (High Availability) ถ้าระบบบางส่วนดาวน์ ระบบที่เหลือก็ยังทำงานได้
*   **Scalability:** สามารถขยายระบบได้รวดเร็วเพื่อรองรับข้อมูลที่ไม่มีโครงสร้างชัดเจนจำนวนมหาศาล

---

## 2. NoSQL แตกต่างจาก SQL อย่างไร?
*   **SQL (Relational):** เก็บข้อมูลแบบมีตาราง มีความสัมพันธ์กัน (Joined tables) เหมาะกับข้อมูลที่มีแบบแผนชัดเจน แต่มีข้อจำกัดในการขยายระบบ เพราะส่วนใหญ่การขยายต้องเป็นวง **Vertical Scaling** (เพิ่มสเปค CPU/RAM ให้เซิร์ฟเวอร์ทื่อๆ ซึ่งมีขีดจำกัด)
*   **NoSQL (Non-relational):** กำจัดปัญหาการทับซ้อนของการ Join ตารางทิ้งไป และถูกออกแบบมาให้ทำ **Horizontal Scaling** (การเพิ่มจำนวนเซิร์ฟเวอร์ให้ช่วยกันทำงาน) ได้ง่ายและสมบูรณ์แบบกว่าตั้งแต่วันแรก เพื่อรองรับปริมาณการใช้งานที่มากขึ้น

## 3. ประเภทของฐานข้อมูล NoSQL (Types of NoSQL)
ฐานข้อมูล NoSQL มีแบ่งย่อยออกไปอีก 4-5 รูปแบบหลัก ตามความเหมาะสมของงาน:

1.  **Key-value store:** รูปแบบที่เรียบง่ายที่สุด เก็บข้อมูลเป็นคู่ Key และ Value (คล้าย Dictionary) เหมาะใช้สำหรับทำ Caching หรือ เก็บข้อมูลชั่วคราวอย่างตะกร้าสินค้าของ User (User Sessions) 
    *   *ตัวอย่าง:* Redis, Memcached
2.  **Document store:** เก็บข้อมูลเป็นเอกสาร (เอกสาร 1 ชิ้น = 1 Record) มักเป็นฟอร์แมต JSON, XML ข้อมูลแต่ละชิ้นไม่ต้องมีฟิลด์เหมือนกันเป๊ะ ทำให้ยืดหยุ่นต่อการเพิ่มฟิลด์ใหม่ๆ ในอนาคต เหมาะกับระบบโปรไฟล์ หรือ Content Management (CMS)
    *   *ตัวอย่าง:* MongoDB
3.  **Wide-column store:** เก็บข้อมูลแยกในรูปแบบคอลัมน์ (Column-oriented) แทนที่จะเป็นแถว (Row-oriented) ระบบดึงเฉพาะคอลัมน์ที่ต้องการโดยไม่ต้องโหลดคอลัมน์อื่นมาให้หนักเครื่อง เหมาะกับงานวิเคราะห์ข้อมูลใหญ่ๆ (Big data analytics)
    *   *ตัวอย่าง:* Apache Cassandra, HBase
4.  **Graph store:** เก็บข้อมูลในรูปแบบจุด (Nodes) และเส้นความสัมพันธ์ (Edges) เหมาะสำหรับระบบที่ข้อมูลทุกอย่างเกี่ยวโยงกันเป็นตาข่ายซับซ้อน เช่น Social Network (ใครเป็นเพื่อนกับใคร), แนะนำสินค้า (Recommendation)
    *   *ตัวอย่าง:* Neo4j
5.  **In-memory store:** เก็บข้อมูลลงในหน่วยความจำหลัก (RAM) ของเซิร์ฟเวอร์ทั้งหมด แทนที่จะเขียนลงฮาร์ดดิสก์ ทำให้การดึงข้อมูลทำงานได้ความเร็วระดับสายฟ้าแลบ

## 4. ข้อดีหลักของ NoSQL (Advantages of NoSQL)
*   **Cost-effectiveness (ประหยัดค่าใช้จ่าย):** ขยายระบบแบบ Horizontal ด้วยฮาร์ดแวร์ธรรมดาๆ หลายเครื่องได้ ประหยัดกว่าการต้องซื้อ Database Server ระดับไฮเอนด์ตัวเดียว
*   **Flexibility (ยืดหยุ่น):** เมื่อไม่ต้องฟิกซ์ Schema ล่วงหน้า การปรับแก้ไขโค้ด อัปเดตฟีเจอร์ใหม่ๆ ก็ทำได้รวดเร็ว (Agile Development) ไม่มีสะดุด
*   **Replication (ทำสำเนาอัตโนมัติ):** ด้วยกลไกทำสำเนาไปเก็บหลายๆ เซิร์ฟเวอร์ ทำให้ระบบพร้อมใช้งานเสมอ และป้องกันข้อมูลสูญหาย
*   **Speed (ทำงานรวดเร็ว):** ตอบโจทย์ Web Application หรือ Mobile App สมัยใหม่ที่ความเร็วในการโหลดข้อมูลส่งผลต่อความพึงพอใจโดยตรง

## 5. Use Cases และ Microservices
ยุคนี้โปรเจกต์ขนาดใหญ่มักไม่ได้ใช้ฐานข้อมูลตัวเดียวคุมทั้งระบบ (Monolithic) แต่ใช้สถาปัตยกรรม **Microservices** (แยกฟีเจอร์เป็นเซิร์ฟวิสย่อยๆ) 

ซึ่ง NoSQL ช่วยสนับสนุนแนวคิดที่ว่า **"ใช้ฐานข้อมูลที่เหมาะสมที่สุดสำหรับงานนั้นๆ" (Polyglot Persistence)** เช่น ระบบตะกร้าสินค้าใช้ Redis (Key-value), ส่วนการเก็บประวัติแชทใช้ MongoDB (Document) ทำให้ไม่ต้องพยายามฝืนเอาทุกอย่างไปยัดไว้ใน SQL ตัวเดียวเหมือนในอดีต
