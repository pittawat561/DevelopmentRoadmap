ภาษาโปรแกรม C# ช่วยให้คุณสามารถสร้างแอปพลิเคชันได้หลายประเภท เช่น:

- แอปพลิเคชันทางธุรกิจสำหรับรวบรวม วิเคราะห์ และประมวลผลข้อมูล
- เว็บแอปพลิเคชันแบบไดนามิกที่สามารถเข้าถึงได้จากเว็บเบราว์เซอร์
- เกม ทั้งแบบ 2D และ 3D
- แอปพลิเคชันด้านการเงินและวิทยาศาสตร์
- แอปพลิเคชันบนคลาวด์ (Cloud-based applications)
- แอปพลิเคชันบนมือถือ

แต่คุณจะเริ่มต้นเขียนแอปพลิเคชันได้อย่างไร?

แอปพลิเคชันทั้งหมดประกอบขึ้นจากโค้ดหลายบรรทัดที่ทำงานร่วมกันเพื่อให้งานสำเร็จลุล่วง วิธีที่ดีที่สุดในการเรียนรู้การเขียนโค้ดคือ การลงมือเขียนโค้ด ดังนั้นจึงขอแนะนำให้คุณเขียนโค้ดตามแบบฝึกหัดในโมดูลนี้และโมดูลอื่นๆ ในเส้นทางการเรียนรู้นี้ การเขียนโค้ดด้วยตัวคุณเองในแต่ละแบบฝึกหัดและสนุกไปกับความท้าทายเล็กๆ ด้านการโค้ด จะช่วยเร่งการเรียนรู้ของคุณให้เร็วขึ้นเยอะเลยครับ

คุณจะได้เริ่มเรียนรู้แนวคิดพื้นฐานทีละน้อยและต่อยอดแนวคิดเหล่านั้นด้วยการฝึกฝนและสำรวจอย่างต่อเนื่อง

ในโมดูลนี้ คุณจะได้:

- เขียนโค้ดภาษา C# บรรทัดแรกของคุณ
- ใช้สองเทคนิคที่แตกต่างกันในการแสดงข้อความ (Print a message) ออกทางหน้าจอ
- วินิจฉัยข้อผิดพลาด (Diagnose errors) เมื่อเขียนโค้ดไม่ถูกต้อง
- ระบุองค์ประกอบไวยากรณ์ (Syntax elements) ของ C# เช่น ตัวดำเนินการ (Operators), คลาส (Classes) และ เมธอด (Methods)

เมื่อจบโมดูลนี้ คุณจะสามารถเขียนโค้ด C# เพื่อแสดงข้อความไปยังส่วนแสดงผลมาตรฐานของคอนโซล (Standard output of a console) ได้ เช่น Windows Terminal โค้ดไม่กี่บรรทัดเหล่านี้จะเปิดประตูบานแรกให้คุณได้สัมผัสกับโครงสร้างของภาษา C# และให้ข้อมูลเชิงลึกที่ประเมินค่าไม่ได้เลยล่ะครับ

ในแบบฝึกหัดภาคปฏิบัตินี้ คุณจะได้ใช้ C# เพื่อเขียนวลีสุดฮิตตลอดกาลของโปรแกรมเมอร์ให้แสดงบนหน้าจอคอนโซล

## เขียนโค้ดบรรทัดแรกของคุณ
มีธรรมเนียมปฏิบัติอันยาวนานในหมู่นักพัฒนาซอฟต์แวร์ นั่นคือการพิมพ์คำว่า "Hello World!" ลงในหน้าต่างแสดงผลบนคอนโซล ดังที่คุณจะได้สัมผัส คุณสามารถเรียนรู้มากมายเกี่ยวกับการเขียนโปรแกรมและภาษา C# จากแบบฝึกหัดง่ายๆ นี้

## พิมพ์โค้ดลงใน C# Code Editor
มี C# Code Editor ให้ใช้งานได้โดยตรงบนเบราว์เซอร์ เครื่องมือ Editor นี้มีสภาพแวดล้อมที่ใช้งานง่าย น้ำหนักเบา สำหรับการเขียน รัน และทดสอบโค้ด C# โดยไม่ต้องติดตั้งซอฟต์แวร์ใดๆ ในคอมพิวเตอร์ของคุณ

เปิด C# Code Editor ในหน้าต่างหรือแท็บเบราว์เซอร์ใหม่

คุณสามารถจัดวางหน้าต่างเบราว์เซอร์ไว้ข้างๆ กัน เพื่อให้สามารถดูคำแนะนำเหล่านี้ไปพร้อมกับตอนที่คุณทำงานใน C# Code Editor ได้

จากนั้นทำการพิมพ์โค้ดนี้ให้เหมือนทุกประการลงใน C# Code Editor:

```csharp
Console.WriteLine("Hello World!");
```

เดี๋ยวคุณจะได้เห็นคำอธิบายว่ามันทำงานอย่างไรและทำไมมันถึงทำงานได้แบบนั้น แต่ก่อนอื่น คุณควรจะได้สัมผัสกับการรันมันก่อน และตรวจสอบให้แน่ใจว่าคุณป้อนมันถูกต้อง โดยคุณจะต้องทำการรัน (Run) โค้ดของคุณ

> **หมายเหตุ**  
> คุณอาจรู้สึกอยากเลือก Copy แล้ว Run ไปเลยเพื่อที่จะไม่ต้องพิมพ์เอง อย่างไรก็ตาม การพิมพ์โค้ดด้วยตัวคุณเองมีประโยชน์หลายอย่างครับ การพิมพ์โค้ดเองช่วยตอกย้ำความจำและความเข้าใจ ซึ่งจะช่วยให้คุณรับรู้ข้อมูลเชิงลึกบางอย่างที่คุณอาจจะไม่ได้รับหากคุณแค่คัดลอกมาวาง

## รันโค้ดแรกของคุณ
กดปุ่ม ▶️ **Run**

ปุ่ม Run จะทำหน้าที่สองอย่าง:

1. ทำการคอมไพล์ (Compile) โค้ดของคุณให้อยู่ในรูปแบบที่สามารถทำงานได้เบื้องหลัง ซึ่งเป็นแบบที่คอมพิวเตอร์สามารถเข้าใจได้
2. ดำเนินการรันแอปพลิเคชันที่คอมไพล์แล้วของคุณ และเมื่อเขียนถูกต้อง มันจะแสดงผล "Hello World!" ออกมา

## สังเกตผลลัพธ์ของคุณ
ในส่วนคอนโซลผลลัพธ์ (Output console) ให้สังเกตผลลัพธ์ของโค้ดของคุณ คุณควรได้ผลลัพธ์ดังนี้:

**Output**
```
Hello World!
```
## ทำอย่างไรหากคุณได้รับข้อความแจ้งข้อผิดพลาด (What to do if you get an error message)
การเขียนโค้ดภาษา C# เป็นแบบฝึกหัดที่ต้องใช้ความแม่นยำ หากคุณพิมพ์ตัวอักษรแม้แต่ตัวเดียวผิดไป คุณจะได้รับข้อความแจ้งข้อผิดพลาดในพื้นที่แสดงผลเมื่อคุณรันโค้ด

ตัวอย่างเช่น หากคุณพิมพ์ตัว c พิมพ์เล็กในคำว่า console ผิดพลาดดังนี้:

```csharp
console.WriteLine("Hello World!");
```

คุณจะได้รับข้อความข้อผิดพลาดต่อไปนี้:

**Output**
```
Error summary:
  CS0103: The name 'console' does not exist in the current context
```

## ข้อความข้อผิดพลาดนี้หมายความว่าอย่างไร?
C# เป็นภาษาแบบ Case-sensitive หมายความว่าตัวอักษรพิมพ์เล็กและพิมพ์ใหญ่มีผลต่างกัน คอมไพเลอร์ C# มองคำว่า `console` กับ `Console` แตกต่างกันเสมือนกับคำว่าแมวกับหมา บางครั้งข้อความแห่งข้อผิดพลาดก็อาจจะสื่อความหมายที่ทำให้สับสนได้ คุณจำเป็นต้องเข้าใจเหตุผลที่แท้จริงเบื้องหลังข้อผิดพลาด ซึ่งคุณจะได้เรียนรู้เกี่ยวกับการใช้ไวยากรณ์ (Syntax) ที่หลากหลายกว่านี้ต่อไป

ในทำนองเดียวกัน หากคุณใช้เครื่องหมาย Single-quotation marks (`'`) แทนเพื่อครอบข้อความ (Literal string) `Hello World!` แบบนี้:

```csharp
Console.WriteLine('Hello World!');
```

คุณจะได้รับข้อความข้อผิดพลาดต่อไปนี้:

**Output**
```
Error summary:
  CS1012: Too many characters in character literal
```

ข้อความพวกนี้คือสิ่งที่สามารถใช้หาคำตอบหรือสืบหาปัญหา (Clue) ว่าเกิดปัญหาที่ไหน อย่างไรก็ตาม ข้อความข้อผิดพลาดนั้นหมายความว่าอย่างไร? "character literal" คืออะไรกันแน่? ในภายหลัง คุณจะได้เรียนรู้เกี่ยวกับข้อมูลตามตัวอักษรประเภทต่างๆ อย่างละเอียด ดังนั้นควรระมัดระวังเมื่อป้อนโค้ดของคุณ

โชคดีที่ข้อผิดพลาดเหล่านี้ไม่ใช่ข้อผิดพลาดถาวรครับ คุณเพียงแค่สังเกตเห็นข้อผิดพลาดนั้น แก้ไขมัน แล้วรันโค้ดของคุณซ้ำอีกครั้งก็พอ

หากคุณพบข้อผิดพลาดขณะที่คุณรันโค้ดตัวเอง ใช้เวลาสักครู่พิจารณาข้อมูลพวกนั้นอย่างใกล้ชิดและตรวจสอบตัวอักษรแต่ละตัวดู และอย่าลืมตรวจสอบให้แน่ใจว่าได้ป้อนโค้ดบรรทัดนี้ลงไปอย่างถูกต้อง ครบถ้วน

> **หมายเหตุ**  
> เครื่องมือตรวจข้อผิดพลาดระหว่างเขียน (Code Editor) จะเฝ้าติดตามโค้ดที่คุณเขียนอยู่อย่างต่อเนื่องครับ โดยการจำลองการทำงานก่อนทำการรันจริง (Pre-compilation) เพื่อตรวจหาข้อผิดพลาดที่อาจเกิดขึ้น ซึ่งทำงานเหมือนกับเครื่องตรวจไวยากรณ์และจะเน้นด้วยขีดเส้นใต้หยักๆ สีแดง เพื่อชี้เป้าว่าบรรทัดไหนที่มีข้อผิดพลาด

ข้อผิดพลาดทั่วไปที่โปรแกรมเมอร์มือใหม่มักจะทำ ได้แก่:

- พิมพ์ตัวอักษรพิมพ์เล็กแทนตัว C พิมพ์ใหญ่ในคำว่า `Console` หรือ W หรือ L ในคำว่า `WriteLine`
- พิมพ์เครื่องหมายจุลภาค (Comma) ในตำแหน่งที่ควรพิมพ์จุด (Period) ระหว่าง `Console` และ `WriteLine`
- ลืมใช้เครื่องหมาย Double-quotation marks หรือใช้ Single-quotation marks แทนในกลุ่มคำว่า `Hello World!`
- ลืมเครื่องหมายเซมิโคลอน (;) ไว้ที่ส่วนท้ายของคำสั่ง

ข้อผิดพลาดแต่ละอย่างเหล่านี้จะขัดขวางไม่ให้โค้ดของคุณถูกคอมไพล์เพื่อรันได้อย่างถูกต้อง เครื่องมือ Code editor จะเน้นข้อผิดพลาดในขั้นตอน Pre-compilation เพื่อช่วยให้คุณระบุและแก้ไขข้อผิดพลาดในขณะที่คุณกำลังพัฒนาโค้ดของคุณเอง ซึ่งคุณสามารถเปรียบเทียบเสมือนเครื่องตรวจสอบตัวสะกดที่ช่วยคุณแก้ไขข้อผิดพลาดในการใช้ไวยากรณ์หรือการสะกดคำในเอกสารของคุณได้เลย

เรามาลองสมมติว่าคุณประสบความสำเร็จในขั้นตอนก่อนหน้านี้ด้วยดี เรามาต่อกันเลยครับ

## แสดงข้อความแบบใหม่ (Display a new message)
ในขั้นตอนนี้ คุณจะได้ทดลองทำให้แสดงจุดไข่ปลา (Comment out) เพื่อป้องกันไม่ให้คำสั่งก่อนหน้าทำงาน จากนั้นจึงเพิ่มคำสั่งที่เขียนใหม่ต่อท้ายด้วยข้อความ .NET Editor ในรูปแบบคำสั่งใหม่

ให้ดัดแปลง (Modify) โค้ดใหม่ที่เขียน ให้บรรทัดนั้นนำหน้าด้วย Slash คู่ (`//`) เช่นนี้:

```csharp
// Console.WriteLine("Hello World!");
```

คุณสามารถทำการเพิ่มหมายเหตุ (Code comment) เพื่อสั่งให้ระบบทำเป็นคอมเมนต์แล้วข้ามไป ให้บรรทัดนี้ได้รับการละเว้นโดยคอมไพเลอร์ด้วยเครื่องหมาย Slash สองตัวพร้อมกัน (`//`)

Code comments เป็นส่วนช่วยเหลือ เมื่อคุณยังไม่ต้องการลบคอมเมนต์ออกทันที แต่ต้องการละเว้นเพื่อซ่อนการทำงานเอาไว้ในขณะนั้น คุณยังสามารถใช้ประโยชน์ในการบอกความต้องการต่อตัวเอง และให้คนอื่นๆ ที่อ่านรู้เพื่อช่วยเตือนว่าโค้ดเหล่านี้ทำหน้าที่อะไร

เมื่อเข้าใจเรียบร้อย จัดการเพิ่มข้อความโค้ดชุดใหม่ให้ตรงตามข้อมูลโค้ดต่อไปนี้:

```csharp
Console.Write("Congratulations!");
Console.Write(" ");
Console.Write("You wrote your first lines of code.");
```

กดปุ่ม ▶️ **Run** อีกครั้ง ครั้งนี้คุณควรได้รับข้อความแบบนี้:

**Output**
```
Congratulations! You wrote your first lines of code.
```
## ความแตกต่างระหว่าง Console.Write และ Console.WriteLine
การเพิ่มโค้ดใหม่สามบรรทัดนี้แสดงให้เห็นถึงความแตกต่างระหว่างเมธอด `Console.WriteLine()` และ `Console.Write()`

`Console.WriteLine()` ทำหน้าที่พิมพ์ข้อความลงในคอนโซลผลลัพธ์ โดยเมื่อจบบรรทัด มันจะเพิ่มคำสั่งขึ้นบรรทัดใหม่ (Line feed) ลงไปท้ายข้อความ คล้ายกับการกดปุ่ม Enter หรือ Return เพื่อสร้างบรรทัดใหม่

หากต้องการพิมพ์ข้อความลงในคอนโซลผลลัพธ์ แต่ไม่ต้องเพิ่มบรรทัดใหม่ต่อท้าย ให้ใช้เทคนิคที่สอง คือ `Console.Write()` ดังนั้นการเรียกใช้ `Console.Write()` ครั้งถัดไปจึงเป็นการพิมพ์ข้อความอื่นลงในบรรทัดเดียวกัน

```csharp
Console.Write("You wrote your first lines of code.");
```

กดปุ่ม ▶️ **Run** อีกครั้ง ครั้งนี้คุณควรได้รับผลลัพธ์ต่อไปนี้:

**Output**
```
Congratulations!
You wrote your first lines of code.
```

โค้ดนี้ช่วยสาธิตความแตกต่างระหว่างสองเมธอด การขึ้นบรรทัดใหม่จะถูกนำมาต่อท้ายด้วย `Console.WriteLine()` ในขณะที่ `Console.Write()` จะพิมพ์ผลลัพธ์ในบรรทัดปัจจุบัน

## ภาษาโปรแกรมคืออะไร?
ภาษาโปรแกรมอย่าง C# ให้คุณสามารถเขียนคำสั่งที่คุณต้องการให้คอมพิวเตอร์ดำเนินการ (Execute) แต่ละภาษาโปรแกรมมีไวยากรณ์ (Syntax) ในแบบของตัวเอง แต่อย่างไรก็ตาม หลังจากที่คุณเริ่มต้นเรียนรู้ภาษาแรกและพยายามจะเรียนรู้ภาษาอื่นๆ คุณจะตระหนักได้ว่าพวกมันทั้งหมดต่างมีแนวคิดร่วมที่คล้ายคลึงกัน งานหลักๆ ของภาษาโปรแกรมคือการเปิดโอกาสให้มนุษย์สามารถถ่ายทอดเจตนาในวิธีการที่มนุษย์สามารถอ่านและทำความเข้าใจ คำสั่งต่างๆ ที่คุณเขียนออกมาในแต่ละภาษานั้นเรียกว่า "ซอร์สโค้ด (Source code)" หรือโดยปกติคือ "โค้ด" ผู้เขียนโค้ดคือผู้พัฒนาซอฟต์แวร์นั่นเอง

ณ จุดนี้ โปรแกรมเมอร์สามารถทำการอัปเดตและเปลี่ยนแปลงโครงสร้างของชุดข้อมูลภายในได้ แต่คอมพิวเตอร์ไม่สามารถเข้าใจข้อมูลดิบที่ส่งเข้ามาได้เลย ดังนั้นโค้ดจะต้องถูกคอมไพล์เป็นรูปแบบข้อมูล (Format) ก่อนที่คอมพิวเตอร์จะสามารถเข้าใจได้

## การคอมไพล์ (Compilation) คืออะไร?
โปรแกรมพิเศษที่ชื่อว่า คอมไพเลอร์ (Compiler) จะแปลงซอร์สโค้ดของคุณเป็นรูปแบบอื่นที่หน่วยประมวลผลกลาง (CPU) ของคอมพิวเตอร์สามารถรันหรือสั่งให้ทำงานได้ เมื่อคุณใช้ปุ่ม ▶️ **Run** ในหน่วยก่อนหน้านี้ โค้ดที่คุณเขียนจะถูกคอมไพล์และถูกสั่งใช้งาน (Executed)

ทำไมโค้ดถึงยังต้องถูกคอมไพล์? ถึงแม้ว่าภาษาโปรแกรมแปลกๆ ส่วนใหญ่จะดูซับซ้อนในตอนแรก แต่มนุษย์ก็สามารถเข้าใจได้ง่ายกว่าภาษาที่คอมพิวเตอร์เลือกใช้ ซีพียูเข้าใจคำสั่งที่แสดงโดยการเปิดหรือปิดสวิตช์เล็กๆ นับแสนหรือล้านตัว คอมไพเลอร์จะเชื่อมสะพานระหว่างโลกสองใบนี้โดยการแปลคำสั่งที่มนุษย์อ่านได้เป็นชุดคำสั่งที่คอมพิวเตอร์เข้าใจ

## ไวยากรณ์ (Syntax) คืออะไร?
กฎสำหรับการเขียนโค้ด C# เรียกว่า **ไวยากรณ์ (Syntax)** เช่นเดียวกับที่ภาษามนุษย์มีกติกาและกระบวนการในเรื่องโครงสร้างรูปประโยค ภาษาโปรแกรมบนคอมพิวเตอร์เองก็มีกำหนดการ (Rules) เช่นเดียวกัน กฎเหล่านี้จะกำหนดคำหลัก (Keywords) และตัวดำเนินการ (Operators) ของภาษา C# และวิธีที่พวกมันถูกนำมาประกอบเข้าด้วยกันจนเป็นโปรแกรม

เมื่อคุณเขียนโค้ดใน .NET Editor คุณอาจสังเกตเห็นการเปลี่ยนแปลงเล็กๆ น้อยๆ ของสีคำต่างๆ ในเครื่องหมายและสัญลักษณ์ (Syntax highlighting) เป็นคุณลักษณะที่มีประโยชน์ที่คุณจะเริ่มใช้เพื่อช่วยค้นหาความผิดพลาดในโค้ดของคุณได้อย่างง่ายดายที่ไม่เป็นไปตามโครงสร้างและกฎที่ควรมีใน C#

## โค้ดของคุณทำงานได้อย่างไร?
มาโฟกัสที่เขียนโค้ดบรรทัดต่อไปนี้:

```csharp
Console.WriteLine("Hello World!");
```

เมื่อคุณทำการรัน (Run) โค้ดที่คุณเคยรันแล้ว คุณจะเห็นว่าตัวอักษร "Hello World!" ถูกพิมพ์ออกมาบนคอนโซลที่ใช้แสดงผล เมื่อวลีนี้ถูกครอบด้วยเครื่องหมายอัญประกาศคู่ (Double-quotation marks) ในโค้ด C# เรียกว่า ตัวอักษรดั้งเดิม (Literal string) ซึ่งมีความหมายตรงตามตัวอักษรแบบเจาะจงที่ได้ถูกระบุเอาไว้เลย ไม่ว่าจะเป็นอักษร C, h, e, l, l, o และอื่นๆ ที่ถูกส่งออกมายืนยันบนหน้าต่างที่แสดงผล

คำว่า `Console` เรียกว่า **คลาส (Class)** โดยคลาสจะเปรียบเสมือน "เจ้าของ" ซึ่งมี **เมธอด (Methods)**; หรือพูดกันภาษาทั่วไป เมธอดจะสามารถอาศัยได้แค่ภายในพื้นที่ของตัวเอง การอ้างถึงเมธอดแต่ละที่ คุณสมควรที่จะต้องรู้ด้วยว่าตอนนี้มันมีสถานะเป็นคลาสประเภทอะไรไปแล้ว ในตอนนี้ ให้นึกถึงคลาสที่เป็นวิธีการนำเสนอแทนคุณสมบัติของอ็อบเจ็กต์ ในกรณีนี้ ทุกการทำงาน (Methods) ที่ทำอยู่บนคอนโซลแสดงผลของคุณนั้น มันถูกสร้างบรรจุอยู่ใน `Console class`

นอกจากนี้ยังมี **จุด (Dot หรือ Period)** ที่จะทำหน้าที่แยกระหว่างชื่อของคลาส `Console` สำหรับกระบวนการเมธอดของตัวเอง `WriteLine()` จุดนี้จัดว่าเป็นตัวดำเนินการเข้าถึงสมาชิกระบบ (Member access operator) หรืออีกนัยหนึ่ง รหัสจุดพวกนี้จะเป็นการ "นำทาง (Navigate)" จากระดับการใช้งานสำหรับคลาสส่วนย่อยนั้นไปเป็นอีกหนึ่งในเมธอดของมัน

ส่วนที่เรียกว่า `WriteLine()` คือ **เมธอด (Method)** คุณมองเห็นเมธอดนึงได้ทุกเวลาผ่านโครงสร้างของส่วนนั้นเนื่องจากมันมักจะมีชุดสัญลักษณ์วงเล็บเปิดปิดอยู่ด้านหลัง เสมือนเป็นตัวบอกการทำงานที่ต้องปฏิบัติของแต่ละจุด เมธอด `WriteLine()` ทำหน้าที่เขียนข้อความแสดงบรรทัดลงในคอนโซลส่งข้อมูล (Output console) ข้อมูลที่สามารถสั่งงานจะถูกทำส่งข้อมูลลงกลางระหว่างวงเล็บเปิดปิด ซึ่งตั้งเป็นตัวแปรพารามิเตอร์ของอินพุต (Input parameter) เมธอดทั่วไปต้องการการรับข้อมูล แต่เมธอดอื่นๆ ก็อาจจะไม่ต้องการ หากคุณมีเป้าประสงค์ที่จะขอยืมรหัสมาใช้กับเมธอด (Invoke) คุณจำเป็นต้องใส่ตัวเครื่องหมายวงเล็บปิดท้ายเสมอเมื่อนำทางชื่อสำหรับเมธอดนึงๆ ชุดวงเล็บทั้งสองตัวถือเป็นองค์ประกอบไวยากรณ์ส่วนท้ายเพื่อดำเนินการเรียกเมธอด

สุดท้ายนี้ เครื่องหมาย **เซมิโคลอน (;)** ทำหน้าที่ประกาศเพื่อจบการทำงาน (End of statement operator) คำสั่งการแบบครบวงจร 1 เส้นทางนับเป็นประโยคคำสั่งใน C# เซมิโคลอนบ่งบอกการจบส่งรหัสเพื่อแจ้งให้ผู้รวบรวมข้อมูลทำงานทราบว่าคุณได้เข้าสู่โหมดการตั้งค่าคำสั่งครบตามที่ต้องการแล้ว

อย่าเพิ่งกังวลหากแนวคิดและคำศัพท์เหล่านี้ยังทำให้ไม่เข้าใจ สำหรับตอนนี้ สิ่งที่คุณต้องจำให้ขึ้นใจเลยก็คือ เมื่อเวลาใดก็ตามที่อยากสั่งพิมพ์ภาพหรือให้ทำงานบนพื้นที่ส่วนการจัดการที่ออกมาให้คอมพิวเตอร์ของคุณ

- ใช้ `Console.WriteLine("Your message here");` 
- ให้เขียนขึ้นต้นด้วยอักษรตัว C ในคำว่า `Console`, และ W ใน `Write`, และ L ใน `Line` เสมอ
- เลือกใช้เครื่องหมายที่ให้ถูกต้องในเรื่องของช่องทางการสื่อสารเพราะนั่นหมายถึงว่ามันคือสิ่งที่ถูกกำหนดสิทธิพิเศษของมันไว้ในโครงสร้างการทำงาน
- หากมีความผิดพลาดเกิดขึ้น ให้ลองสังเกตกวาาดจุดบอดนั้น หาเจอเมื่อไหร่แล้วลองสั่งให้มันทำงานใหม่อีกซักทีนะครับ

> **เคล็ดลับ**  
> สร้างกระดาษชดระยะเวลาไว้เตือนใจ (Cheat sheet) หรือสรุปรหัสสั้นๆ แล้วก็จดทุกคำพูดที่ได้เขียนไปเพื่อการประมวลผลที่ไวขึ้นหากคุณเจอเรื่องใหม่ๆ ไว้เตือนตัวเอง

## ทำความเข้าใจกระแสโฟลว์การทำงาน (Flow of execution)
เป็นเรื่องสำคัญที่จะเข้าใจโฟลว์ทำงานของโปรแกรม นี่หมายความว่าคำสั่งในสคริปต์แบบเขียนโค้ดเอาไว้จะเริ่มปะติดปะต่อตามลำดับทีละบรรทัดๆ ไปเรื่อยๆ จนจบการทำงานได้โดยไม่มีอะไรที่ถูกสั่งหรือเขียนออกมาไว้ให้ทำอีก ข้อมูลคำสั่งบางบรรทัดอาจบอก CPU ของเครื่องเพื่อยื้อเวลาไว้ ในการขอหน่วงเวลาก่อนที่เริ่มไปต่อ คำสั่งประเภทอื่นอาจถูกใช้วางตัวแทรกเพื่อปรับเปลี่ยนโฟลว์การทำงานได้

ตอนนี้ก็ถึงตาที่จะมาทบทวนสิ่งที่คุณได้เรียนรู้ ทุกโมดูลมีแบบทดสอบง่ายๆ หากคุณติดขัด คุณสามารถกลับมามองทบทวนจากจุดที่ได้รับไปได้เลย ลองเริ่มไปทบทวนในโมดูลถัดไปเพื่อนำความเป็นตัวเองใช้ใน C# ดูกัน

เป้าหมายของคุณคือการเขียนโค้ดที่แสดงข้อความง่ายๆ ไปยังคอนโซลผลลัพธ์ ในขณะเดียวกันก็ทำความคุ้นเคยกับไวยากรณ์ (Syntax) ไปด้วย คุณได้เขียนโค้ดบรรทัดแรกเริ่มโดยใช้ไวยากรณ์ C# พื้นฐาน คุณได้เรียนรู้เทคนิคสองวิธีในการแสดงผลข้อมูลตามตัวอักษร (Literal-string) ออกไปยังคอนโซล คุณยังได้เรียนรู้ด้วยว่าจะต้องมองหาอะไรเมื่อพบข้อผิดพลาดในโค้ดของคุณ และสุดท้าย คุณสามารถระบุองค์ประกอบไวยากรณ์ของ C# เช่น คลาสและเมธอด รวมถึงวัตถุประสงค์ของสัญลักษณ์พิเศษหลายตัวที่เป็นที่รู้จักกันในนามตัวดำเนินการ (Operators) ได้ ถือว่าคุณได้เริ่มต้นก้าวแรกสู่การสร้างแอปพลิเคชันที่มีความซับซ้อนมากขึ้นเรียบร้อยแล้ว
