# How does the Internet Work? (อินเทอร์เน็ตทำงานอย่างไร)

> 🌐 **แหล่งที่มา:** [cs.fyi — How does the Internet Work?](https://cs.fyi/guide/how-does-internet-work)

---

## 1. อินเทอร์เน็ตคืออะไร? (Introduction)

อินเทอร์เน็ตคือ **"เครือข่ายของเครือข่าย" (A network of networks)**
ก่อนที่จะมีอินเทอร์เน็ต เรามีเครือข่ายระดับเล็กๆ (Local Network) เช่น คอมพิวเตอร์หลายเครื่องในบ้าน หรือในบริษัทที่เชื่อมต่อกัน เมื่อนำเครือข่ายย่อยๆ เหล่านี้ทั่วโลกมาเชื่อมต่อเข้าด้วยกัน จึงเกิดเป็นอินเทอร์เน็ตขึ้นมา

*   **จุดกำเนิด:** เริ่มต้นในช่วงปลายยุค 1960 โดยกระทรวงกลาโหมสหรัฐฯ เพื่อสร้างระบบสื่อสารแบบกระจายศูนย์ (Decentralized) ที่สามารถทนทานต่อการโจมตีได้
*   **ปัจจุบัน:** เป็นโครงสร้างพื้นฐานทั่วโลก ควบคุมด้วยโปรโตคอลมาตรฐาน

---

## 2. คำศัพท์และแนวคิดพื้นฐาน (Basic Concepts)

Чтобыเข้าใจการทำงานของอินเทอร์เน็ต ต้องรู้จักคำศัพท์เหล่านี้:

*   **Packet:** หน่วยข้อมูลขนาดเล็ก อินเทอร์เน็ตจะหั่นข้อมูลใหญ่ๆ เป็นชิ้นเล็กๆ เรียกว่า Packet เพื่อส่งไปตามสาย (หรือคลื่น)
*   **Router (เราเตอร์):** อุปกรณ์ที่ทำหน้าที่เป็น "จราจร" คอยส่ง Packet ข้อมูลข้ามไปยังเครือข่ายต่างๆ จนกว่าจะถึงปลายทาง
*   **IP Address:** หมายเลขประจำเครื่อง (เช่น `192.168.1.1`) ที่ระบุตัวตนของแต่ละอุปกรณ์ในเครือข่าย
*   **Domain Name:** ชื่อที่มนุษย์อ่านเข้าใจง่าย (เช่น `google.com`) เพื่อใช้แทน IP Address ของเว็บไซต์
*   **DNS (Domain Name System):** ระบบสมุดหน้าเหลืองที่แปลจาก `Domain Name` ให้กลับกลายเป็น `IP Address`
*   **HTTP/HTTPS:** โปรโตคอลหลักที่ใช้รับ-ส่งข้อมูลระหว่าง Browser (Client) กับ Website (Server)

---

## 3. ความสำคัญของโปรโตคอล (Protocols)

โปรโตคอลคือ **"กฎและมาตรฐาน"** ที่ทำให้อุปกรณ์ต่างยี่ห้อ ต่างระบบปฏิบัติการ สามารถคุยกันรู้เรื่อง

*   **IP (Internet Protocol):** รับผิดชอบการหาเส้นทางและส่ง Packet ไปให้ถึงปลายทางที่ถูกต้อง
*   **TCP / UDP:** 
    *   **TCP (Transmission Control Protocol):** รับประกันว่าข้อมูลต้องส่งถึงมือผู้รับครบถ้วนและเรียงลำดับอย่างถูกต้อง
    *   **UDP (User Datagram Protocol):** เน้นส่งเร็ว ไม่รับประกันว่าของจะถึงหรือครบถ้วน (เหมาะกับดูไลฟ์สด หรือเล่นเกม)
*   **HTTP (Hypertext Transfer Protocol):** โปรโตคอลสำหรับจัดการการส่งข้อมูลเว็บเพจ รูปภาพ ไฟล์ต่างๆ

---

## 4. การจัดการ IP Address และ Domain Name

1.  **IP Address:** ทำหน้าที่บอกว่าอุปกรณ์นี้ "อยู่ที่ไหน" บนเครือข่าย หากไม่มี IP Address ก็ส่งของให้ไม่ได้
2.  **Domain Name:** เนื่องจากมนุษย์จำตัวเลข IP ยาวๆ ไม่เก่ง จึงใช้ชื่อโดเมนแทน
3.  **กระบวนการ DNS:**
    เมื่อคุณพิมพ์ชื่อเว็บในเบราว์เซอร์ → คอมพิวเตอร์จะส่งคำถามไปยัง **DNS Server** ว่า "โดเมนนี้มี IP อะไร?" → DNS ตอบกลับเป็นหมายเลข IP → คอมพิวเตอร์นำหมายเลข IP นั้นไปติดต่อไปหาเว็บเซิร์ฟเวอร์

---

## 5. พื้นฐานการพัฒนาแอปด้วย TCP/IP

เมื่อทำแอปพลิเคชันหรือเว็บที่ดึงข้อมูลผ่านอินเทอร์เน็ต (TCP/IP) จะต้องเข้าใจสิ่งเหล่านี้:

*   **Ports (พอร์ต):** หมายเลขที่ระบุ "แอปพลิเคชัน" หรือ "บริการ" ปลายทางในเครื่องเดียวกัน (เช่น พอร์ต 80 สำหรับ HTTP, พอร์ต 443 สำหรับ HTTPS)
*   **Sockets (ซ็อกเก็ต):** เป็นการรวมกันของ `IP Address + Port` เพื่อสร้างจุดสิ้นสุด (Endpoint) ในการสื่อสาร
*   **Connections (การเชื่อมต่อ):** การตกลงกันระหว่าง 2 Sockets ว่าจะเริ่มคุยกันแล้วนะ (เช่น กระบวนการ Handshake)

---

## 6. ความปลอดภัยด้วย HTTPS และ SSL/TLS

**HTTPS (HTTP Secure)** คือ HTTP เวอร์ชั่นที่ปลอดภัยกว่า โดยเข้ารหัสข้อมูลที่รับ-ส่งผ่านโปรโตคอล **SSL/TLS**

*   **Certificates (ใบรับรอง):** ใช้ยืนยันตัวตนของเซิร์ฟเวอร์ ออกโดยองค์กรที่น่าเชื่อถือ (CA) ป้องกันการถูกสวมรอย
*   **Handshake:** ก่อนเริ่มส่งข้อมูล Client และ Server จะคุยกันเพื่อตกลงวิธีการเข้ารหัส (Encryption Algorithm)
*   **Encryption (การเข้ารหัส):** ข้อมูลจริงจะถูกเข้ารหัส ทำให้คนกลางที่แอบดักจับข้อมูล (Hacker/ISP) อ่านไม่รู้เรื่อง

---

## 7. เทรนด์ที่จะมาพัฒนาอินเทอร์เน็ตในอนาคต

*   **5G:** ความเร็วสูงขึ้น ความหน่วงต่ำลง
*   **IoT (Internet of Things):** อุปกรณ์รอบตัวจะเชื่อมต่ออินเทอร์เน็ตและคุยกันเองได้เยอะขึ้น
*   **AI (Artificial Intelligence):** ช่วยประมวลผลระบบและมอบประสบการณ์ในการใช้อินเทอร์เน็ตที่ดีขึ้น
*   **Blockchain:** ความปลอดภัยระบบบัญชีแยกประเภทแบบกระจายศูนย์
*   **Edge Computing:** นำการประมวลผลไปไว้ใกล้ผูัใช้งาน (Edge) มากขึ้น แทนที่จะวิ่งไปที่เซิร์ฟเวอร์ศูนย์กลางไกลๆ ช่วยลดความหน่วง (Latency) ได้มหาศาล

> 💡 **สรุปสั้นๆ:** อินเทอร์เน็ตคือคอมพิวเตอร์นับพันล้านเครื่องที่ต่อสายคุยกัน โดยใช้กฎ (Protocols) ชุดเดียวกัน ในการหั่นข้อมูล (Packets), หาเส้นทาง (Routing) และนำไปประกอบร่างที่ปลายทางอย่างถูกต้องครับ!
